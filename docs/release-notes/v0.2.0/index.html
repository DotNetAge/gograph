    v0.2.0¶

    All notable changes to the GoGraph project will be documented in this file.

    The format is based on Keep a Changelog, and this project adheres to Semantic Versioning.

    [v0.2.0] - 2026-03-29¶

    Added¶

    Parser Refactoring¶

    • New Lexer (pkg/cypher/lexer/)
    • Complete OpenCypher lexical analysis implementation
    • Support for all keywords, operators, and literals
    • Improved error reporting with line/column position information

    • Support for single-line and multi-line comments

    • New Parser (pkg/cypher/parser/)

    • Recursive descent parser implementation
    • Full OpenCypher syntax support
    • Correct operator precedence handling

    • Parameterized query support ($param syntax)

    • AST Refactoring (pkg/cypher/ast/)

    • Modular AST node definitions
    • Visitor pattern support (visitor.go)

    • Complete position tracking

    • Observability Support (pkg/cypher/observability*.go)

    • Tracing interface
    • Metrics interface

    • No-op default implementations

    • Context Management (pkg/cypher/context/)

    • Query execution context

    • Variable scope management

    • Utility Functions (pkg/cypher/utils/)

    • ExprToValue() - Expression to value conversion
    • ExprToString() - Expression to string conversion

    Testing¶

    • OpenCypher compliance test suite (compliance_test.go)
    • Boundary condition tests (boundary_test.go)
    • Lexer tests
    • Parser tests

    Documentation¶

    • OpenCypher syntax reference (opencypher.md)
    • Compliance test report (COMPLIANCE_REPORT.md)
    • Code audit report (CODE_AUDIT_REPORT.md)
    • Code quality report (CODE_QUALITY_REPORT.md)

    Changed¶

    Architecture Optimization¶

    • Executor Refactoring (executor.go)
    • Extracted withTx common method for unified transaction handling
    • Extracted handleMatchDelete method to eliminate duplicate code

    • Reduced code lines by approximately 60 lines

    • Creator Optimization (creators/create.go)

    • Replaced duplicate code with utility functions

    • Improved MERGE statement handling

    • Modifier Optimization (modifiers/update.go)

    • Replaced duplicate code with utility functions

    • Unified expression conversion logic

    • Matcher Optimization (matchers/match.go)

    • Replaced duplicate code with utility functions
    • Improved variable-length path queries

    Resource Management¶

    • Adjacency List Fix (graph/adjacency.go)
    • Fixed iterator resource leak issue
    • Used defer to ensure proper resource release

    API Improvements¶

    • Database Interface (api/db.go)
    • Uses new parser entry point
    • Improved transaction handling flow

    Fixed¶

    • Fixed iterator not properly closed in GetRelatedNodes causing resource leak
    • Fixed undirected relationship query support
    • Fixed variable-length path query issues
    • Fixed GetNode returning incorrect ID value
    • Fixed int vs int64 type mismatch issues

    Breaking Changes¶

    File Structure Changes¶

    • Removed old parser implementation:
    • pkg/cypher/parser.go → pkg/cypher/parser/

    • pkg/cypher/ast/ast.go → pkg/cypher/ast/ (multiple files)

    • Removed old test files:

    • pkg/cypher/parser_test.go
    • pkg/cypher/executor_test.go
    • pkg/cypher/creators/create_test.go
    • pkg/cypher/matchers/match_test.go
    • pkg/cypher/modifiers/update_test.go

    API Changes¶

    • Parser entry point change: 

      // Old version
import "github.com/DotNetAge/gograph/pkg/cypher"
ast, err := cypher.Parse(query)

// New version
import "github.com/DotNetAge/gograph/pkg/cypher"
ast, err := cypher.NewParser(query).Parse()

    • AST structure changes:

    • Node types split from single file to multiple modular files
    • Added visitor pattern interface

    Removed¶

    • Old parser implementation (pkg/cypher/parser.go)
    • Old AST definition (pkg/cypher/ast/ast.go)
    • Deprecated test files
    • V2-related naming and files

    Performance¶

    • Eliminated approximately 140 lines of duplicate code
    • Unified transaction handling logic reduces memory allocations
    • Lexer uses strings.Builder for optimized string concatenation
    • Iterator resource management optimization prevents memory leaks
