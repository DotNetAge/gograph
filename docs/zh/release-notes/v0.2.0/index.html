    v0.2.0¶

    本文件记录了 GoGraph 项目的所有重要变更。

    格式基于 Keep a Changelog， 本项目遵循 语义化版本。

    [0.2.0] - 2026-03-29¶

    新增特性¶

    新解析器架构¶

    • 词法分析器模块 (pkg/cypher/lexer/)
    • 完整的OpenCypher词法分析
    • 支持所有关键字、运算符和字面量

    • 改进的错误报告机制，包含行列位置信息

    • 语法解析器模块 (pkg/cypher/parser/)

    • 递归下降解析器实现
    • 支持完整的OpenCypher语法
    • 操作符优先级正确处理

    • 支持参数化查询 ($param 语法)

    • 抽象语法树重构 (pkg/cypher/ast/)

    • 模块化的AST节点定义
    • 访问者模式支持 (visitor.go)

    • 完整的位置信息追踪

    • 可观测性支持 (pkg/cypher/observability*.go)

    • Tracer接口用于分布式追踪
    • Meter接口用于指标收集

    • No-op默认实现

    • 工具函数包 (pkg/cypher/utils/)

    • ExprToValue() - 表达式转值

    • ExprToString() - 表达式转字符串

    • 验证器模块 (pkg/cypher/visitor/)

    • AST验证访问者
    • 函数注册表支持分类

    • 打印器用于AST可视化

    • 上下文包 (pkg/cypher/context/)

    • 查询执行上下文
    • 变量作用域管理

    改进优化¶

    架构改进¶

    • 从单体解析器迁移到模块化架构
    • 将词法分析器、语法解析器和AST分离为独立包
    • 改进代码组织，遵循Go最佳实践

    代码质量¶

    • 提取公共工具函数，消除约60行重复代码
    • 添加 withTx 辅助方法处理事务
    • 修复 GetRelatedNodes 中的资源泄漏，使用defer关闭迭代器
    • 改进错误处理，使用自定义错误类型

    性能优化¶

    • 词法分析器使用 strings.Builder 优化字符串构建
    • 减少热路径中的内存分配

    问题修复¶

    资源管理¶

    • 修复 adjacency.go 中的迭代器资源泄漏
    • 确保正确使用defer进行资源清理

    类型处理¶

    • 修复int/int64类型不匹配问题
    • 改进属性值编码

    移除内容¶

    废弃代码¶

    • 移除旧的单体解析器 (pkg/cypher/parser.go)
    • 移除旧的AST实现 (pkg/cypher/ast/ast.go)
    • 移除过时的测试文件（已迁移到新结构）

    不兼容变更¶

    API变更¶

    • Parse() 函数现在返回 *ast.Query 而非旧的AST类型
    • Executor方法更新为使用新的AST结构
    • Parser和AST包的导入路径已更改

    迁移指南¶

    // 旧版本
import "github.com/DotNetAge/gograph/pkg/cypher"
result, err := cypher.Parse(query)

// 新版本
import "github.com/DotNetAge/gograph/pkg/cypher/parser"
p := parser.New(query)
ast, err := p.Parse()

    合规性报告¶

    总体统计： - ✅ 支持: 45项 (88.2%) - ⚠️ 部分支持: 5项 (9.8%) - ❌ 不支持: 1项 (2.0%)

    不支持的功能： - UNION子句

    部分支持的功能： - WHERE子句 - 字符串匹配（正则表达式不支持） - 关系模式 - 关系方向（<- 简写语法不支持） - 函数 - 聚合函数（COUNT(*) 语法不支持） - 基础语法 - 标识符命名规则（$ 符号不支持） - 高级特性 - 列表推导（模式推导语法不支持）

    完全支持的功能： - 基础语法（关键字大小写、注释） - 数据类型（标量、时间、复合类型） - 节点模式（标签、属性、变量） - 关系模式（类型、变长路径） - CRUD操作（MATCH/CREATE/MERGE/SET/DELETE/REMOVE/UNWIND） - WHERE子句（比较、逻辑、空值判断） - RETURN子句（排序、分页、表达式） - WITH子句（聚合、过滤） - 表达式（算术、属性访问、CASE） - 函数（聚合、数学、字符串、列表、图函数） - 高级特性（复杂查询）

    代码质量审计¶

    第一阶段：无价值代码清理¶

    扫描结果： | 类别 | 发现数量 | 处理结果 | | ------------- | -------- | ---------------------- | | 空结构/空方法 | 3处 | ✅ 保留（设计模式需要） | | 死代码 | 0处 | ✅ 无发现 | | 未使用常量 | 0处 | ✅ 全部在使用中 |
